Основные фазы сборки проекта

  1. compile
    2. Компилирование проекта
  2. test
    3. Тестирование с помощью JUnit тестов
  3. package
    4. Создание .jar файла или war, ear в зависимости от типа проекта
  4. integration-test
    5. Запуск интеграционных тестов
  5. install
    6. Копирование .jar (war , ear) в локальный репозиторий $link
  6. deploy
    7. публикация файла в удалённый репозиторий

К примеру нам нужно создать jar проекта. Чтобы его создать набираем: 

mvn package
Но перед тем как создаться jar файл будут выполняться все предыдущие фазы compile и test , а фазы integration-test, install, deploy не выполнятся. Если набрать 
mvn package
то выполнятся все приведённые выше фазы.

Особняком стоит фаза clean удаление всех созданных в процессе сборки артефактов: .class, .jar и др. файлов. Если она специально не указана команде запуска то она не выполняется.
B и фаза site создание документации Т. к . команда mvn понимает когда ему передают несколько фаз то для сборки проекта "с нуля" выполняют: 

mvn clean package

С полным списком фаз и их описанием (на английском языке) можно ознакомиться здесь http://maven.apache.org/guides/introduction/introduction-to-the-lifecycle.html#Lifecycle_Reference
